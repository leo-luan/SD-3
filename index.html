Definição


A organização de SD’s trata, em grande parte, dos componentes de SW que constituem o sistema. Essas arquiteturas de software os dizem como os vários componentes de SW devem ser organizados e como devem interagir.

Estilos Arquitetônicos


Definem como os componentes estão conectados uns aos outros, os dados trocados entre eles, e por fim, a maneira como são configurados.
  • Componente é uma unidade modular com interfaces bem definidas, sendo substituíveis dentro do sistema.

Estilos

  • Em camadas: os componentes são organizados em camadas, e um componente de uma camada Li, tem permissão de chamar componentes na camada Li-1, mas não o contrário.
  • Baseado em objetos: cada objeto é um componente, e se comunicam por meio de chamadas de métodos remotos.

Estilos

  • Centrada em dados: processos se comunicam em torno de um repositório de dados comum.
  • Baseada em eventos: processos se comunicam por meio da propagação de eventos (queue ou pub/sub).
